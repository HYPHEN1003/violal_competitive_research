# AI 市場リサーチパイプライン — 「アイデアを出させるな、市場を調べさせろ」

## 原則
- AIにアイデアを聞かない（同じプロンプト = 同じ回答 = レッドオーシャン）
- AIにはデータを読ませ、パターンを見つけさせ、人間が判断する材料を揃えさせる
- 儲かる事業はゼロから発明しない。既に儲かっている領域を**ズラすか絞る**

## 前提入力
ユーザーに確認:
1. **検討している領域**: どんな市場/業界を考えているか
2. **自分の強み**: 経験・スキル・人脈・既存の資産
3. **目的**: 新規事業 / 既存事業の拡張 / クライアント向け提案

---

## 5ステップ段階承認フロー

### Step 1: 競合の広告分析（15分）

**やること:**
- Meta Ad Library（facebook.com/ads/library）で領域のキーワードを検索
- 3ヶ月以上出稿されている広告をピックアップ（長期出稿 = 利益が出ている証拠）
- 広告のスクリーンショット/テキストをユーザーに渡してもらう

**AIが分析すること:**
- フック（冒頭の訴求）のパターン分類
- ターゲット層の特定
- 訴求している課題と解決策
- 共通メッセージ vs 差別化メッセージ

**出力:** 競合メッセージマップ（誰が / 誰に / 何を言っているか）

**→ ユーザー承認**

### Step 2: 顧客の不満を大量分析（20分）

**方法A: レビュー分析**
- 競合プロダクトの低評価レビュー（1-3つ星）を収集
- Amazon / App Store / note / Google Maps 等から
- AIに投入し「繰り返し出現する不満テーマ」を抽出

**方法B: SNS/コミュニティ分析**
- Reddit / X / Yahoo!知恵袋 / 業界フォーラムの投稿を収集
- liteparse でPDF資料があればテキスト化

**注目キーワード:**
- 「〜があればいいのに」「〜がほしい」→ 未充足の需要
- 「〜にうんざり」「〜が使いにくい」→ 既存への不満
- 「〜の方法がわからない」「〜で困っている」→ 情報ギャップ

**出力:** 不満ランキング（出現頻度順）

**→ ユーザー承認**

### Step 3: ギャップ特定 = 市場の穴（10分）

**核心の問い:**
> 「競合の広告やコンテンツでは訴求されていないが、顧客が繰り返し不満や要望として挙げているテーマは何か？」

Step 1（競合が言っていること）× Step 2（顧客が求めていること）を突き合わせる。

**出力:** ギャップリスト（競合カバーなし × 需要あり = 参入ポイント候補）

**→ ユーザー承認**

### Step 4: 自分の強みとの交差点（15分）

**ニッチは見つけるものではなく、アングルで作る。**

ユーザーに壁打ち（/grill-me と連携可能）:
1. 「あなたの経験の中で、他の人には真似できないことは？」
2. 「Step 3のギャップリストの中で、あなたが一番語れるテーマは？」
3. 「同じ市場で、他と全く違うアプローチで成功している人はいるか？」

**出力:** アングルの言語化
> 「○○の経験を持つ人が、○○の切り口で、○○の層に向けて提供する」

**→ ユーザー承認**

### Step 5: 小さく出して検証する設計（10分）

いきなり商品を作らない。コンテンツで反応を見る。

**設計内容:**
- X / note / LinkedIn で5-10本のテスト投稿案を作成
- 各投稿が Step 4 のアングルの異なる側面をカバー
- 2-3週間で投稿 → 反応データをAIで分析

**分析指標:**
- どの投稿が最もエンゲージメントが高かったか
- コメントで最も多かった質問（= 人がお金を払ってでも知りたいこと）
- 想定外の層からの反応

**出力:** テスト投稿プラン + 検証基準

## 出力ファイル

```
docs/research/
├── {date}-{market}-competitor-map.md    ← 競合メッセージマップ
├── {date}-{market}-customer-pain.md     ← 顧客不満ランキング
├── {date}-{market}-gap-analysis.md      ← ギャップ分析
├── {date}-{market}-angle.md             ← アングル定義
└── {date}-{market}-test-plan.md         ← テスト投稿プラン
```

## 4つの参入パターン（参考）
1. **ズラす** — ターゲット、価格帯、UI、地域のいずれかを変える
2. **絞る** — 大きな市場からニッチを切り取る
3. **弱みを突く** — DXが遅れている業界、マーケ力で勝てる斜陽産業
4. **課題を解く** — 自分や身近な人の課題（最初の顧客がいる状態でスタート）

## やってはいけないこと
- AIの出力を鵜呑みにする（データは材料、判断は人間）
- リサーチだけで終わる（Step 5まで到達しないリサーチはただの知的娯楽）
- ツール選びに時間を使う（何を聞くかが100倍重要）
