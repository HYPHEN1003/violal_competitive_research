# 営業提案パイプライン — 4時間で商談対応の提案書を完成

## 依存スキル（~/.claude/skills/ に要インストール）
- **mkt-sales-playbook**（Stage 2 で使用）
- **mkt-content-ops**（Stage 4 で使用）
- インストール: `bash ~/Documents/dev/hyphen-marketing-skills/install.sh`
- リポジトリ: https://github.com/HYPHEN1003/hyphen-marketing-skills

## 原則
**「速くして高く売る」** — 2週間かかっていた提案書を4時間で出す。
クライアントにとっての価値は同じ以上。だから単価を下げる理由がない。

## 前提入力
ユーザーに以下を確認:
1. **クライアント名**: 誰に提案するか
2. **課題/ニーズ**: クライアントが困っていること
3. **提案内容**: 何を提案するか（概要だけでOK）
4. **予算感**: おおよその予算レンジ
5. **商談日**: いつまでに必要か

---

## 5ステージパイプライン

### Stage 1: 仮説構築（15分）
**使用スキル: `/hypothesis-driven-analysis`**

- ゴール1行を設定: 「○○社に、△△の導入を合意してもらう」
- 開始仮説を立てる
- Key Questions 5つを定義
- バックアップ仮説2つを用意

**→ ユーザー承認**

### Stage 2: 商談準備ブリーフ（15分）
**使用スキル: `/mkt-sales-playbook`**

- クライアント企業のリサーチ
- 競合状況の把握
- 価格パッケージの設計（3段階: Basic / Standard / Premium）
- 想定される異議と対応策

**→ ユーザー承認**

### Stage 3: 提案スライド設計（30分）
**使用スキル: `/create-proposal-slide`（3ステップ段階承認方式）**

- Step 1: ゴール + 各ページの主張を並べる
- Step 2: 各ページのボディ要素を設計
- Step 3: テンプレートにデータを挿入
- 「作らせるな、埋めさせろ」原則を徹底

**→ ユーザー承認**

### Stage 4: 品質チェック（15分）
**使用スキル: `/mkt-content-ops`（Expert Panel）**

- 7-10名の仮想エキスパートパネルが提案書を採点
- 90点未満なら自動改善ループ（最大3ラウンド）
- スコア表 + 改善済みコンテンツを出力

**→ ユーザー最終承認**

### Stage 5: 最終仕上げ（15分）

- Speaker Notes の追加
- クライアントの業界用語への調整
- 「だから何？」の最終チェック
- 出力: Markdown + （必要なら）Marp でスライド化

---

## 出力

```
docs/proposals/
├── {date}-{client}-proposal.md      ← 提案書本体
├── {date}-{client}-briefing.md      ← 商談準備ブリーフ
├── {date}-{client}-hypothesis.md    ← 仮説分析
└── {date}-{client}-score.md         ← Expert Panel採点結果
```

## 時間配分

| ステージ | 時間 | 人間の作業 |
|---------|------|-----------|
| 仮説構築 | 15分 | 仮説の承認 |
| 商談準備 | 15分 | 価格パッケージの承認 |
| スライド設計 | 30分 | 主張と「だから何？」を考える |
| 品質チェック | 15分 | 最終承認 |
| 最終仕上げ | 15分 | 微調整 |
| **合計** | **90分** | 判断と承認のみ |

## ルール
- 人間が「何を言うか」を決める。AIは「データを埋める」だけ
- 各ステージの境界を明確にし、承認なしで先に進まない
- 引き算の原則: ゴールに直結しない情報は入れない
