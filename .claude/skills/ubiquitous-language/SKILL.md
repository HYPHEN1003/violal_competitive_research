# ユビキタス言語

現在の会話からドメイン用語を抽出・形式化し、一貫した用語集を作成して `UBIQUITOUS_LANGUAGE.md` に保存する。

## プロセス

1. **会話をスキャン** — ドメインに関連する名詞・動詞・概念を特定
2. **問題を検出**:
   - 同じ単語が異なる概念に使われている（曖昧性）
   - 異なる単語が同じ概念に使われている（同義語）
   - 曖昧またはオーバーロードされた用語
3. **正規用語集を提案** — 推奨用語を選定
4. **`UBIQUITOUS_LANGUAGE.md` に書き出し**
5. **会話内にサマリーを出力**

## 出力形式

```markdown
# ユビキタス言語

## [グループ名]

| 用語 | 定義 | 避けるべき別名 |
|------|------|---------------|
| **顧客** | 商品を購入する個人または法人 | クライアント, 取引先, アカウント |

## 関係性

- **顧客** は複数の **注文** を持つ
- **注文** は1つ以上の **請求書** を生成する

## フラグされた曖昧性

- 「アカウント」が **顧客** と **ユーザー** の両方の意味で使われていた
```

## ルール
- **推奨用語を選べ。** 複数の単語が同じ概念を指す場合、最適な1つを選び、他は「避けるべき別名」にする
- **定義は1文以内。** それが「何であるか」を定義する（何をするかではない）
- **ドメイン用語のみ。** 汎用プログラミング用語（配列、関数等）は含めない
- **再実行時は差分更新。** 既存ファイルを読み、新規は (new)、変更は (updated) を付ける
