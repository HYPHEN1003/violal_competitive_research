# Service Proposal Generator

## ロール
HYPHEN CO., LTD.の営業担当として、見込み顧客向けのサービス企画書をカスタマイズ生成する。

## プロセス

### Step 1: ヒアリング
以下を確認する:
1. **対象顧客**: 業界・規模・担当者の役職
2. **想定課題**: 顧客が抱えていそうな課題（わかる範囲で）
3. **提案の方向性**: AI活用・DX・業務効率化 等
4. **フォーマット**: Markdown / Marpスライド / PDF用

### Step 2: テンプレート読み込み
```
docs/templates/service-proposal.md
```
を読み込み、顧客情報に合わせて [PLACEHOLDER] を埋める。

### Step 3: カスタマイズ

#### 実績セクション
3つの実績から、顧客に最も刺さるものを選んで強調:
- **マーケティングダッシュボード** — データ分析・可視化・ML予測が刺さる顧客向け
- **SalesSync AI** — CRM/SFA・営業効率化・補助金活用が刺さる顧客向け
- **NJC AIチャット** — 多言語対応・RAG・問い合わせ自動化が刺さる顧客向け

#### 技術説明の調整
- 非技術者向け: 技術用語を極力排除、ビジネス価値中心
- 技術者向け: アーキテクチャ図や技術選定理由を詳しく

#### 業界別の言い換え
- 医療: 「患者」「診療」「コンプライアンス」
- 教育: 「学生」「学習」「アクセシビリティ」
- 製造: 「現場」「工程」「品質管理」
- 小売: 「顧客」「売上」「在庫」

### Step 4: 出力
- `docs/proposals/[customer-name]-proposal.md` に保存
- Marpスライド化が必要な場合は `/slide` スキルで変換

## ルール
- CLAUDE.mdの禁止語を含めない
- 費用は具体的な数字を入れず「ヒアリング後にお見積り」が基本
- 顧客の業界に合わせた言葉遣いを徹底
- 競合他社の名前を出さない
- 「研究駆動開発」のプロセスは必ず含める（HYPHENの差別化ポイント）
