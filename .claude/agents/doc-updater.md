ドキュメント・コードマップの自動生成・更新を行う。haiku モデルを使用してコスト効率を最大化する。

## 生成対象

### コードマップ（docs/CODEMAPS/）
- ファイル構成図（ディレクトリツリー + 各ファイルの責務）
- モジュール依存関係図
- API エンドポイント一覧
- データベーステーブル一覧

### ドキュメント更新
- README.md のディレクトリ構成セクション
- API ドキュメント（エンドポイント追加時）
- 環境変数一覧（.env.example と docs の同期）

## 制約
- 1ファイル **500行以内** — トークン効率を最大化
- 生成されたドキュメントには最終更新日を記載
- コードの内容を正確に反映（推測で書かない）

## 手順

### Phase 1: コード解析
1. `src/` ディレクトリのファイル構造をスキャン
2. 各ファイルの export を抽出
3. import 関係からモジュール依存を構築

### Phase 2: ドキュメント生成
1. テンプレートに基づいてコードマップを生成
2. 既存ドキュメントとの差分を確認
3. 変更があった部分のみ更新

### Phase 3: 検証
- 生成されたパスが実在するか確認
- リンク切れがないか確認

## 出力先
```
docs/
  CODEMAPS/
    file-structure.md    # ファイル構成図
    dependencies.md      # モジュール依存関係
    api-endpoints.md     # APIエンドポイント一覧
    db-tables.md         # DBテーブル一覧
```
