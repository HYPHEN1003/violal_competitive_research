# エージェント・スキル活用ガイド

## 基本フロー

```
① 要件整理 → ② デザイン → ③ 実装 → ④ レビュー → ⑤ テスト
```

各フェーズで適切なエージェント / スキルを使い分ける。

## エージェント一覧と使い所

| エージェント | 使い所 |
|-------------|--------|
| **designer** | UI/UXデザイン、デザインシステム構築、Figma連携 |
| **ux-writer** | LPコピー、CTA文言、チャットボット会話設計 |
| **prompt-engineer** | Claude APIのプロンプト設計・最適化 |
| **code-reviewer** | PR前のコードレビュー、セキュリティチェック |
| **qa-tester** | テストケース作成、Lighthouse監査、クロスブラウザ確認 |
| **db-architect** | テーブル設計、RLSポリシー、インデックス最適化 |

## スキル使用例

### デザインフェーズ
```
/design-system-architect   → デザイントークン・コンポーネント定義
/brand-identity-creator    → ブランドアイデンティティ構築
/ui-ux-pattern-master      → 画面設計・フロー設計
```

### 実装フェーズ
```
/figma-to-code             → FigmaデザインをコードにURLから変換
/design-to-code            → テキスト記述のデザインをコードに変換
/create-lp-section         → LPセクションの実装
```

### レビューフェーズ
```
/design-critique           → デザインレビュー
/accessibility-auditor     → WCAG 2.2 AA準拠チェック
/review-prompt             → AIプロンプトの品質レビュー
```

## サブエージェント運用ルール

1. **1タスク1サブエージェント** — メインコンテキストを汚さない
2. **調査・探索はサブエージェントに委譲** — コード検索、ファイル確認等
3. **結果はメインに要約して返す** — 全文コピーではなく要点のみ
